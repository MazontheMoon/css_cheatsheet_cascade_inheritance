Conflicting Declarations

Later declarations in the document will overwrite earlier ones (read from top to bottom)

  1. Declarations in user agent style sheets (browser defaults)
  2. Normal declarations in user style sheets(customer style set by user)
  3. Normal declarations in author style sheets (styles set by us – web developers)
  4. Important declarations in author style sheets
  5. Important declarations in user style sheets

Specificity

Specificity is the measure of how specific a selector is – meaning how many elements it could match

  • Element selectors low specificity
  • Class selectors Element selectors
  • ID selectors Class selectors
  • Only !important ID selectors

The amount of specificity a selector has is measured using 4 different values which can be thought of as 1000’s 100’s 10’s and 1’s – 4 single digits in 4 columns

  1. Thousands – score 1 in this column if the declaration is inside a style attribute
  2. Hundreds – score 1 in this column for each ID selector within the overall selector
  3. Tens – score 1 in this column for each class or attribute selector, or pseudo-element within the overall selector
  4. Ones – score 1 in this column for each element of selector, or pseudo-element within the overall selector

So for example:

/* specificity 0113 */
#outer div ul .nav a { }

Inheritance

CSS provides four special universal properties for specifying inheritance

inherit

Sets the property value to a selected element to be the same as that of its parent element

initial

Sets the property value to a selected element to be the same as the value set for that element in the browser’s default style sheet (if no value set by browsers default style sheet and the property is naturally inherited then the property value is set to inherit instead)

unset

Resets the property to its natural value which means if the property is naturally inherited it acts like inherit, otherwise it acts like initial

revert

Reverts the property to the value it would have had if the current origin had not applied any styles to it

Unsurprisingly initial and unset are unsupported in IE
